PrivacyFirst

PrivacyFirst is a data privacy and security initiative for web developers and administrators.

How many horror stories can we think of from just the past few years involving invasive tracking practices, massive data breaches the size of entire countries, and lawsuits that threaten the state of things like device encryption and personal data control, just to name a few?

In this new, evolved World Wide Web, where almost any website or service has the ability to recieve and retain any information from users that they want, as well as the ability to track them from site to site without their knowledge, it is time for us — the web developers and administrators of the world — to start placing the user's privacy and security first, before anything else.

When the topic of the prioritization of data privacy and security comes up, many times it devolves into diatribes and rants about "profitibility" and "feasibility," among other things. What if we could instill change for data security and privacy while also retaining profit margins and doing it with ease? A lot of the unease about having to move a company or service towards a more privacy-positive direction comes from not knowing alternatives to the services and behaviors that are causing security concerns in the first place. That's what the PrivacyFirst initiative is for.

Below, you'll find a non-comprehensive list of various commonly-used services that don't respect user privacy or have shady/unknown security practices, along with better alternatives.

"Heap is an analytics platform that helps product, marketing, and customer success teams craft exceptional digital experiences that convert and retain users." — Heap's website.

Feature List Pros & Cons

  • APIs for multiple platforms & frameworks.
  • Robust free plan fit for most small-scale sites.
  • Better performance on web when using the Javascript API.
  • Documented security features and policies.
  • Dashboard setup can be confusing for beginners.
  • Analytic user identity deletion feature for the web can be difficult to setup.
  • Some features can be a bit bloated for most purposes.
  • Plans do not have clear pricing. Pricing only available on inquiry.
  • Uses Amazon Web Services for their data hosting servers.1

Privacy Features

Heap allows admins to create and delete user identities. All data sent to Heap from your site is encrypted with TLS. All stored data on Heap's AWS servers are encrypted at rest. Heap is SOC 2 and GDPR compliant. Heap conducts biannual pen tests of their infrastructure. Heap allows customization to avoid the capture of Personally Identifiable Information (PII.) Heap can allow the admin to disable all text capture and prevent data collection where required. Heap has a documented Data Protection Officer (DPO.) Heap has no recorded data breaches.

Notes

1 While this isn't nessesarily a bad thing, Amazon is a corporation known for their deceptive or lackluster data security and privacy policies, and have had data leaks in the past from their storage services.
